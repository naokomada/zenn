久しぶりに Openclaw のアップデートをしたら、approval機能がついていました。

その後、何をやってもapprovalを求められるようになり、たまにうまく承認を受信できず、しまいにagent自身もキレ始めるという意味の分からない事態になります。

キレたagentに、humanのほうで実行してくれとコマンドを渡されたタイミングで、真面目に権限設定を考えることにしました。

## やりたかったこと

やりたかったのは、ざっくり言うと次の3つ。

- agentには、ある程度自律的に他のアプリとかコマンドを実行して作業してほしい。
- ただし、ホスト全体を自由に触れる状態にはしたくない。
- あと複数agentで共有したいファイルやリポジトリがある。

つまり「何でも自由に実行」でも「完全に何もできない」でもなく、限定された実行環境として運用したい。

## 何をやればいいか

ざっくり、やるべきことは、コマンドの実行権限をつけることと、ファイルの閲覧・編集権限をつけることです。

- コマンドの実行権限 Openclawが実行可能なコマンドを設定します。
- ファイルの閲覧・編集権限 Openclawが閲覧・編集できるファイルを、ディレクトリ単位で設定します。

## 今回採用した構成

具体的にやったのはこんな感じです。

### コマンドの実行権限をつける

1. Linux 側で下限境界を作る

まず絶対にOpenclawがやらないでほしいことは、確実な方法（これはOpenclawにバグがあっても確実という意味）で制限しないといけないです。

このためOpenClawは専用ユーザーで動かします。openclaw ユーザーで実行、sudo は与えません。これでコマンドとファイルに基本的なアクセス制限ができます。

2. 個別のコマンドをOpenclawの設定で行う

つぎに個別の操作をOpenclaw側の設定で許可します。

まずは、tools.profile の選定がかなり重要です。

実は最初は tools.profile が messaging になっていました。これだと自分が期待していた workspace 内ファイル操作や docs 閲覧に必要なtoolがほぼ使えません。toolでできることはtoolでやらせるほうがOpenclawの挙動的にわかりやすいため、ここは必要に応じて先に設定します。

自分は coding に変更して、まずは「workspace 内では作業できる」という出発点が作れました。

後は、exec-approvals.json を作成して、個別にコマンドの実行許可をつけていきます。
基本的にはapproveを求められたときに、許可して構わないと思ったコマンドには Allow Always をすることで、exec-approvals.json に追加できます。

ちなみに、ここもtoolでやればいい作業をわざわざコマンドで実行しようとしてapprovalが発生することがあるので、toolを使うように注意することも必要です。


### ファイルの閲覧・編集権限をつける

前提としてOpenclawを専用ユーザーにしているので、自身またはグループが所有しているファイルしか操作できないようにしています。

つぎに、Openclawが機能として持っている、workspaceのみの閲覧制限を使います。

ただこれだけだと、複数agent間で共有してほしいファイルがあったときに、いちいちそのagentのworkspaceにコピーする必要があります。

このため、複数agentで共有したいリポジトリやドキュメントがあったらworkspace外に置きつつ、各 agent の workspace 内に bind mount して見せることにしました。
これだと、agentからは自身のworkspaceのファイルに見えるので操作ができます。

あと、ここでもtoolでなるべく操作させることが重要です。

例えばread toolと、execでfind + headは一見同じことに見えますが、read で workspace 外の絶対パスを読むと sandbox root エラーになる一方、exec は tools.profile と exec-approvals.json 次第で、より広い範囲を見られることがあります。

つまりそもそもworkspaceに閲覧を絞りたかったら、ファイル閲覧のコマンドの実行を絞りつつ、ファイルをみたいときはfs系toolを優先するようにagentに伝えておく必要があります。
この辺はskill化などしていつでも思い出してもらえるようにするのがおすすめです。

## まとめ

今の時点では、最終的には次の整理に落ち着いています。

- Linux 側で下限境界を作る。
- tools.profileの検討をして、toolでできることはちゃんとtoolでやらせる。
- exec-approvals.jsonでコマンド実行は個別に許可。
- workspace内だけファイル閲覧を許可。
- 外部のファイルを共有したかったら bind mount する。
